Palette Studio — How it works

Color Sources

Independent brand colors. All roles inherit their hue from Sources. You can add multiple sources and assign a specific source to each role via the dropdown menu next to the role.

Semantic Roles (groups)

  • Core — Primary and Secondary. The main brand colors.
  • Neutral — Background, Surface, Border, Text. Generated via HCT using the brand hue.
  • Accent — Additional accent colors from other Sources.
  • Semantic — Error, Warning, Success, Info. Fixed hues (red/yellow/green/blue); saturation depends on the brand and Arousal.
  • Extended — Highlight, On Primary, and other supporting roles.

Mood Sliders

  • Valence — Lightness of the palette. Negative = dark theme, positive = light theme.
  • Arousal — Saturation. Negative = muted, dull tones, positive = bright, vivid tones.
  • Temperature — Warm/cool. Negative = cool bluish tint, positive = warm orange tint.
  • Shades — Number of steps in the scale (5–12).

Contrast Check (APCA)

Uses APCA — the WCAG 3.0 algorithm, which is more accurate than the old WCAG 2.1. Displayed for two background colors that can be set in the Contrast Check section.

  • Lc 75+ Normal text of any size
  • Lc 60+ Large text (18px+) or bold text
  • Lc <60 Decorative elements only

Harmony

Automatically arranges Sources on the color wheel. Complementary — opposite color, Triadic — three colors spaced by 120°, etc. Click an active preset again to undo.

From Image

Extracts dominant colors from an image using K-means clustering in OKLCH. Applies them to Sources. Works even for B&W images.

AI Generation

Connects to a local model via LM Studio (port 1234). Describe the mood — the model selects a hue for each Source and adjusts the sliders. The model's response is displayed below the Generate button.

Export

CSS Variables, Tailwind config, or JSON. The ↓ CSS button in the header copies the CSS to the clipboard immediately.
